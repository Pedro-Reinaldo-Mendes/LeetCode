# 268. Missing Number

[LeetCode Problem Link](https://leetcode.com/problems/missing-number/)

## Performance

- **Runtime**: 5 ms (Beats 20.88%)
- **Memory**: 47.40 MB (Beats 60.59%)

## Complexity Analysis

- **Time Complexity**: $O(n)$, where $n$ is the size of the array. We iterate through the array once to insert elements into the `HashSet` ($O(n)$) and then iterate from $0$ to $n$ ($O(n)$) to check for the missing number using $O(1)$ set lookups.
- **Space Complexity**: $O(n)$, to store all the numbers in the `HashSet`.

## Approach

The approach uses a **HashSet** to efficiently check for the presence of each expected number.

1.  **Iterate through the array**: Add every number in `nums` to a `HashSet`.
2.  **Check for existence**: Iterate through the range `[0, n]` (where $n$ is `nums.length`). For each number `i`, check if it's in the set.
3.  **Result**: The first number `i` that is not found in the set is the missing number.

### Solution.java

```java
class Solution {
    public int missingNumber(int[] nums) {
        Set<Integer> set = new HashSet<>();

        for (int num : nums) {
            set.add(num);
        }

        for (int i = 0; i <=  nums.length; i++) {
            if (!set.contains(i)) return i;
        }

        return -1;
    }
}
```
