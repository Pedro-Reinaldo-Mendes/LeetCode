# 383. Ransom Note

[LeetCode Problem Link](https://leetcode.com/problems/ransom-note/)

## Performance

- **Runtime**: 1 ms (Beats 99.90%)
- **Memory**: 46.44 MB (Beats 66.41%)

## Complexity Analysis

- **Time Complexity**: $O(n + m)$, where $n$ is the length of `magazine` and $m$ is the length of `ransomNote`. We iterate through `magazine` once to build the frequency map and then through `ransomNote` once to verify.
- **Space Complexity**: $O(1)$, because the size of the frequency array is fixed at 26 (for the lowercase English letters), independent of the input size.

## Approach

The approach uses a **Frequency Map** (implemented as an array for efficiency) to keep track of available characters in the magazine.

1.  **Count Magazine Characters**: Initialize an integer array of size 26. Iterate through `magazine` and increment the count for each character.
2.  **Verify Ransom Note**: Iterate through `ransomNote`:
    *   For each character, check if its count in the frequency array is greater than 0.
    *   If not, return `false` because we cannot construct the note.
    *   Otherwise, decrement the count and continue.
3.  **Result**: If all characters in the note are successfully processed, return `true`.

### Solution.java

```java
class Solution {
    public boolean canConstruct(String ransomNote, String magazine) {
        int[] arr = new int[26];

        for (char c : magazine.toCharArray()) {
            arr[c - 'a']++;
        }

        for (char c : ransomNote.toCharArray()) {
            if (arr[c - 'a'] == 0) return false;
            arr[c - 'a']--;
        }
        
        return true;
    }
}
```
