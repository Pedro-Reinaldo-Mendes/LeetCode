# LeetCode - 1603. Design Parking System

## Performance

- **Runtime:** 7 ms (Beats 99.90%)

## Complexity Analysis

- **Time Complexity:** $O(1)$
  - Both the constructor and the `addCar` method operate in constant time.
- **Space Complexity:** $O(1)$
  - We use a fixed-size array to store the count of parking lots, regardless of the number of operations or input size.

## Logic Description

The problem requires us to design a parking system with three types of parking spaces: big, medium, and small, with a fixed number of slots for each.

1.  **Data Storage**: We use an integer array `lots` of size 4 to store the available slots for each car type. The car types are 1 (big), 2 (medium), and 3 (small), so mapping them directly to indices 1, 2, and 3 is convenient.
2.  **Initialization**: In the constructor, we set the values for `lots[1]`, `lots[2]`, and `lots[3]` based on the inputs.
3.  **Parking a Car**: When `addCar(carType)` is called:
    - We check if `lots[carType]` is greater than 0.
    - If it is, we decrement the count and return `true` (slot available).
    - If it's 0, we return `false` (no slot available).

This approach is highly efficient as it uses constant time and space for all operations.

### Solution.java

```java
/**
 * 1603. Design Parking System
 *
 * @see <a href="https://leetcode.com/problems/design-parking-system/">Design Parking System</a>
 */
class ParkingSystem {

    int[] lots = new int[4];

    public ParkingSystem(int big, int medium, int small) {
        lots[1] = big;
        lots[2] = medium;
        lots[3] = small;        
    }
    
    public boolean addCar(int carType) {
        if (lots[carType] == 0) return false;
        lots[carType]--;
        return true;
    }
}
```
