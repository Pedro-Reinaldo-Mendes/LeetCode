# 53. Maximum Subarray

[LeetCode Problem Link](https://leetcode.com/problems/maximum-subarray/)

## Performance

- **Runtime**: 1 ms (Beats 99.93%)
- **Memory**: 76.90 MB (Beats 96.86%)

## Complexity Analysis

- **Time Complexity**: $O(n)$, where $n$ is the length of the array. We perform a single pass through the input array.
- **Space Complexity**: $O(1)$, as we only use two integer variables (`maxSum` and `currSum`).

## Approach

The approach uses **Kadane's Algorithm**, an efficient dynamic programming technique for finding the maximum subarray sum.

1.  **Initialize Variables**: Set `maxSum` to the first element of the array and `currSum` to 0.
2.  **Iterate through the Array**:
    *   Add the current element `nums[i]` to `currSum`.
    *   **Update Max**: If `currSum` is greater than `maxSum`, update `maxSum`.
    *   **Reset Current Sum**: If `currSum` drops below 0, reset it to 0. This is because any subarray starting with a negative sum will only decrease the sum of a subsequent subarray.
3.  **Result**: Return the `maxSum`.

### Solution.java

```java
class Solution {
    public int maxSubArray(int[] nums) {

        int maxSum = nums[0];
        int currSum = 0;

        for (int i = 0; i < nums.length; i++) {
            currSum += nums[i];
            if (currSum > maxSum) maxSum = currSum;
            if (currSum < 0) currSum = 0;
        }

        return maxSum;
    }
}
```
