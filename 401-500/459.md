# 459. Repeated Substring Pattern

[LeetCode Problem Link](https://leetcode.com/problems/repeated-substring-pattern/)

## Performance

- **Runtime**: 31 ms (Beats 72.97%)
- **Memory**: 46.54 MB (Beats 91.97%)

## Complexity Analysis

- **Time Complexity**: $O(n^2)$, where $n$ is the length of the string `s`. In the worst case, we check all possible lengths of substrings that are divisors of $n$. For each valid length, we reconstruct the string which takes $O(n)$.
- **Space Complexity**: $O(n)$, to store the reconstructed string in a `StringBuilder`.

## Approach

The approach iterates through all possible lengths of a potential repeating substring and verifies if repeating it forms the original string.

1.  **Divisor Search**: A substring can only repeat to form `s` if its length `l` is a divisor of `s.length()`. We check all lengths from $1$ up to $n/2$.
2.  **String Reconstruction**:
    *   For each valid divisor `l`, extract the prefix of length `l`.
    *   Repeat this prefix $n/l$ times using a `StringBuilder` to build a candidate string.
3.  **Validation**: Compare the candidate string with the original string `s`. If they match, `s` follows a repeated substring pattern.
4.  **Result**: If no divisor length $l$ works, return `false`.

### Solution.java

```java
class Solution {
    public boolean repeatedSubstringPattern(String s) {
        int sLength = s.length();
        for (int l = 1; l <= sLength/2; l++) {
            if (sLength % l != 0) continue;
            String subStr = s.substring(0, l);
            StringBuilder sb = new StringBuilder(subStr);
            for (int r = 1; r < sLength / l; r++) {
                sb.append(subStr);
            }
            if (sb.toString().equals(s)) return true;
        }

        return false;
    }
}
```
