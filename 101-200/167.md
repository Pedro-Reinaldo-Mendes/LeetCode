# 167. Two Sum II - Input Array Is Sorted

[LeetCode Problem Link](https://leetcode.com/problems/two-sum-ii-input-array-is-sorted/)

## Performance

- **Runtime**: 2 ms (Beats 96.38%)
- **Memory**: 48.18 MB (Beats 93.72%)

## Complexity Analysis

- **Time Complexity**: $O(n)$, where $n$ is the length of the input array. In the worst case, we move the pointers a total of $n$ times.
- **Space Complexity**: $O(1)$, as we only use two integer pointers.

## Approach

The approach uses the **Two Pointers** technique, which is optimal for sorted arrays.

1.  **Initialize Pointers**: Set the left pointer `l` at the beginning (0) and the right pointer `r` at the end (`numbers.length - 1`).
2.  **Iterate and Adjust**:
    *   Calculate the sum of elements at `l` and `r`.
    *   If the sum matches the `target`, the solution is found.
    *   If the sum is **less than** the target, increment the left pointer `l` to increase the total sum.
    *   If the sum is **greater than** the target, decrement the right pointer `r` to decrease the total sum.
3.  **Result**: Return the 1-indexed positions of `l` and `r` as required by the problem.

### Solution.java

```java
class Solution {
    public int[] twoSum(int[] numbers, int target) {
        // Let's use two pointers in order to keep space complexity as O(1)

        int l = 0;
        int r = numbers.length - 1;

        while (numbers[l] + numbers[r] != target) {

            if (numbers[l] + numbers[r] < target) l++;
            else r--;
        }

        return new int[]{l + 1, r + 1};
    }
}
```
