# 141. Linked List Cycle

[LeetCode Problem Link](https://leetcode.com/problems/linked-list-cycle/)

## Performance

- **Runtime**: 0 ms (Beats 100.00%)
- **Memory**: 47.02 MB (Beats 10.79%)

## Complexity Analysis

- **Time Complexity**: $O(n)$, where $n$ is the number of nodes in the linked list. In the case of no cycle, the fast pointer reaches the end in $n/2$ steps. In the case of a cycle, the pointers will meet within $n$ steps.
- **Space Complexity**: $O(1)$, as we only use two pointers (`slow` and `fast`).

## Approach

The approach uses **Floyd's Cycle-Finding Algorithm** (also known as the "Tortoise and the Hare").

1.  **Check Edge Cases**: If the head is null or consists of only one/two nodes without a cycle, return `false`.
2.  **Initialize Pointers**: Set the `slow` pointer to the head and the `fast` pointer to `head.next.next`.
3.  **Traverse the List**:
    *   While the `fast` pointer and its next node are not null:
        *   If `fast` and `slow` meet at the same node, a cycle is detected; return `true`.
        *   Move `slow` one step forward (`slow = slow.next`).
        *   Move `fast` two steps forward (`fast = fast.next.next`).
4.  **Result**: If the `fast` pointer reaches the end of the list, no cycle exists; return `false`.

### Solution.java

```java
/**
 * Definition for singly-linked list.
 * class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode(int x) {
 *         val = x;
 *         next = null;
 *     }
 * }
 */
public class Solution {
    public boolean hasCycle(ListNode head) {

        if (head == null || head.next == null || head.next.next == null) return false;
        
        ListNode slow = head;        
        ListNode fast = head.next.next;
        
        while (fast.next != null && fast.next.next != null) {
            if (fast == slow) return true;
            slow = slow.next;
            fast = fast.next.next;
        }

        return false;
    }
}
```
